Text2Pyx

This website creates images from only the URL query string.
This can be useful for applications like Carl-bot tags, where the language is very bare-bones.

How It Works

Generated images are formatted as https://text2pyx.herokuapp.com/generate?w=5&h=5&p=b000fwffff&3h=1&d=bwbwbbwbwb6wb3wbw3b&m=10.

The URL query has 4 mandatory parameters and 2 optional parameters.

The first two parameters are w and h.
These two decide the image's width and height respectively.

The third parameter is p. This decides the palette.
The palette is described using repeated entries of this format:

  • Palette index, one character.
  • Red value of palette color. Two hexadecimal bytes.
  • Green value of palette color. Two hexadecimal bytes.
  • Blue value of palette color. Two hexadecimal bytes.
  • Alpha value of palette color. Two hexadecimal bytes.

An example entry would be BDEADBEEF.
Palette indices cannot be numeric.

The last mandatory parameter is d, which encodes the data of the image.
The data is encoded with palette indices, going from left to right, then top to bottom.
A number after a palette index denotes repeating the palette index that many times.
An example data string would be bwbwbbwbwb6bw3bwb3w.

Optional Parameters

The first optional parameter is m.
This defines the factor that the image is scaled to.
This can be any positive number, e.g. 5.33, as long as it fits in the scale requirements.

The other optional parameter is 3h.
If this is included, the RGBA colors of the palette are only one byte, and the byte is repeated twice. (E621 turns into EE662211)

Limits

Maximum area: 921600 (1280x720)
Images can be generated by the same IP address 8 times a minute.
Images are cached with the query, so redundant generation shouldn't happen.
